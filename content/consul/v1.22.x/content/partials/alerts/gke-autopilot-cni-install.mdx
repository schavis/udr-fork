<Note>

GKE Autopilot clusters do not support installing the Consul CNI plugin.

- The Consul CNI plugin installer requires write access to Kubernetes node-level `hostPath` volumes. This access is required to place the CNI binary in the node CNI bin directory, write CNI configuration under `/etc/cni/net.d`, and reconcile the CNI installation (for example, rotate tokens and update CNI plugin configurations).
- GKE Autopilot restricts `hostPath` volumes in write mode and blocks privileged workloads, so the CNI installation may fail with admission errors such as `autogke-no-write-mode-hostpath` and `autogke-disallow-privilege`. Refer to [Google’s Autopilot security restrictions](https://docs.cloud.google.com/kubernetes-engine/docs/concepts/autopilot-security#built-in-security).
- To configure traffic redirection on GKE Autopilot, use the `connect-inject-init` container at the Pod level with `CAP_NET_ADMIN` privileges. On Autopilot, enable NET_ADMIN using `--workload-policies=allow-net-admin` on the cluster and allow NET_ADMIN in the Pod security context so the init container can configure `iptables`.

</Note>
