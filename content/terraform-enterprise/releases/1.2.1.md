## Improvements
1. Updated git to 2.53.0 built with OpenSSL to support X509 Policy Constraints extension in certificates

## Bug Fixes
1. Atlas configured to use Redis Sentinel without mTLS will now correctly connect to Redis when necessary. Previously Redis connections would fail due to a misconfiguration introduced in v1.1.0, but only when using Redis Sentinel without an mTLS certificate
2. You can now configure an Load Balancer expect 200 from the `/api/v1/health/readiness` endpoint. Previously you would get a 301, unless a Host header matched the TFE_HOSTNAME.
