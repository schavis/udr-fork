# `/sys/rotation/orphans`

@include 'alerts/enterprise-only.mdx'

Use the `/sys/rotation/orphans` endpoints to manage [orphaned](/vault/docs/enterprise/automated-credential-rotation/rotation-policies#orphaned-rotation-entries)
rotation entries for plugins that support automated rotation via the centralized automated rotation system.

See the [automated rotation documentation](/vault/docs/enterprise/automated-credential-rotation/overview)
for more general information on how Vault Enterprise handles orphaned rotation entries.

## Read a list of orphans

Fetch a list of all orphaned rotation entries in Vault Enterprise. We
recommend reviewing orphaned entries for insight into broken/failing rotations.

| Method | Path                           |
| :----- | :----------------------------- |
| `GET`  | `/sys/rotation/orphans/` |

### Sample request

```shell-session
$ curl \
    --header "X-Vault-Token: ${VAULT_TOKEN}" \
    ${VAULT_ADDR}/v1/sys/rotation/orphans
```

### Sample response

```json
{
    "orphans": ["ldap/static-role/mary", "ldap/static-role/alice"]
}
```
