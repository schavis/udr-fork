# transit envelope decrypt

@include 'alerts/enterprise-only.mdx'

The `transit envelope decrypt` command uses the Transit plugin to decrypt data encryption keys stored 
with ciphertext to decrypt files and streams of arbitrary size without reading the plaintext into Vault. 
Decryption happens entirely on the caller side.

The caller must have write permissions for the `decrypt` endpoint for the given key encrypting key.

## Usage

### Command arguments

- `key_path` (string : "")` - Path to the transit key for DEK/EDK generation. 
  Must be in the form `<mount>/keys/<key_name>` where `<mount>` is the path
  to the Transit mount without namespaces and `<key_name>` is the target
  key.
- `file_path (string : "")` The path to a file to decrypt, or `-` to decrypt
  input from stdin.  If decrypting from stdin, there must be only one
  `file_path` input.  Decrypted files will be output to the input filename
  with the suffix removed.

### Command Options

- `--quiet` (bool: false) - Display no output unless there is an error.
- `--output` `(string: "")` - Decrypted output will be written to the
  specified file.  You cannot use `--output` if there is more than one
  input.  If `--output` is supplied, `--suffix` must not be supplied.
- `--suffix` `(string: ".vee") - Suffix to remove from output file names. If
  `--suffix` is supplied, `--output` must not be supplied.
- `--aad` `(string: "") - Additional authenticated data.  This is either a base64  
  encoded binary blob, or is read from a file if specified as @file-name.  Additional 
  authenticated data is added to the authentication present in the output.  
  Decryption will fail if the aad supplied at decrypt is absent or different.

Additional authenticated data must match the data given during the
encryption operation or decryption will fail.

### Example

```shell-script
$ vault transit envelope decrypt -o really-big-file2.zip transit/keys/test-key really-big-file.zip.vee
```
